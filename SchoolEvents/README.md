# Информационна система за организиране и споделяне на училищни събития и инициативи

## Описание
Проектът представлява уеб приложение за управление и публикуване на училищни събития. 
Потребителите могат да разглеждат събития, а администраторите и учителите да създават и управляват съдържание.

## Технологии
- ASP.NET Core MVC
- Entity Framework Core
- SQL Server / LocalDB
- Bootstrap
- JavaScript

## Основни функционалности
- Показване на предстоящи и минали събития
- Детайлна информация за събития
- Добавяне, редактиране и изтриване на събития
- Потребителски роли (Admin, Teacher)
- Календар на събития

## Стартиране на проекта
1. Отворете проекта във Visual Studio.
2. Проверете connection string в `appsettings.json`.
3. Отворете Package Manager Console.
4. Стартирайте приложението.

## Забележка
За да се използва пълната функционалност на проекта, е необходимо базата данни да бъде възстановена на компютъра, на който ще се стартира приложението. Това се извършва чрез предоставения backup (.bak) файл.

Стъпки за възстановяване на базата данни:
1. Инсталирайте Microsoft SQL Server и SQL Server Management Studio (SSMS), ако не са налични на компютъра.
2. Стартирайте SQL Server Management Studio и се свържете към локалния SQL Server.
3. В лявата част, в прозореца Object Explorer, намерете папката Databases.
4. Натиснете с десен бутон върху Databases и изберете Restore Database...
5. В отворения прозорец изберете опцията Device.
6. Натиснете бутона с трите точки (...) и посочете предоставения .bak файл.
7. Потвърдете избора с OK.
8. В полето за име на базата данни проверете дали е изписано правилното име.
9. Натиснете OK, за да започне възстановяването на базата данни.
10. След успешно възстановяване отворете проекта във Visual Studio.
11. Проверете дали в файла appsettings.json е зададен правилният connection string, който сочи към възстановената база данни.
12. Стартирайте приложението.

Важно:
Без възстановяване на базата данни от предоставения backup файл, проектът ще се отвори само като изходен код и няма да съдържа наличните данни, необходими за пълната работа на системата.
